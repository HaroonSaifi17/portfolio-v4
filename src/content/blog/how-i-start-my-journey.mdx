## The Spark

In 2022, right after finishing 12th grade, I was pretty bad at coding even though I had studied C++ for two years. I was not interested in programming, but I was curious about hacking. While exploring that world, I learned that phishing was one of the easiest attack vectors, so I tried to build a demo Instagram login page.

I didn't have a laptop or any real knowledge of HTML or CSS, so I started learning on my phone using **Termux**. Neovim was the only editor that worked well for me, so I learned it from the start. My first phishing page was a copy of the original site with a small change to send data to my Telegram bot, and I was proud that it worked.

## Building Portfolios

That small win pushed me to learn more, so I decided to make a portfolio website. I built it with only HTML and CSS, and making a navbar and carousel from scratch was exciting. Then it got deleted from my phone, and I realized how important version control is. I started using Git immediately.

My next portfolio looked decent with good colors, design, and a parallax effect, but it still felt outdated to me. I wanted it to be more responsive and modern, so I chose **Angular** as my first framework. I picked Angular because I believed choosing harder things would make me learn faster.

I built a new portfolio in Angular 14 and tried to replicate a strong UI/UX design I found online, adding a blog and other features. It turned into a really good learning experience.

## PrepZone and Early Projects

Alongside the portfolio, I started building an edtech website for students — a startup idea from a friend. It was a tough project for a beginner, but bit by bit it took shape. I also built a backend for it using Express and Node. I rewrote the code several times to make it cleaner, which felt like real progress.

Around the same time, I was in my first year of engineering and applied to tech societies, and I got selected in all of them as a tech team member. They were impressed by my portfolio and the PrepZone website, so I also started doing hackathons and building other projects. Not long after, I hit **1000+ commits** on GitHub and felt confident in my stack and web development skills.

## Leveling Up

From there I moved to TypeScript because I realized JavaScript gets messy without types. I learned React, SSR, CSR, and state management while the ecosystem was evolving fast. It was overwhelming, but I managed to absorb it in a few months and started building more complex projects.

I also got my first laptop in my first year of college, so I moved away from Termux. My workflow stayed similar though: WSL with Arch, Neovim, and tmux. I started maintaining my dotfiles and building custom workflows.

## First Internship

Around that time I landed an internship at an early-stage startup and began working with React and Web3. Web3 was new to me, so I started learning it from scratch. It was the beginning of a new chapter — transitioning from a self-taught student building side projects to a developer working on production applications.
